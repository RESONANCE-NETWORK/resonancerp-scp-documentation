ix.quetes

Quest system for mission management and player objectives.

This module manages quests that can be assigned to players with various types of objectives. Quests have types, classes, and can be filtered accordingly.

Functions

ix.quetes.Abandon(client, questID)

 View source »

Abandons an active quest, removing it from the player's active quests.

Triggers the quest's OnCancelled callback if defined and frees assigned resources.

Parameters

  • client table

    The player entity abandoning the quest

  • questID string

    The unique quest identifier

Returns

  • bool

    true if quest was successfully abandoned

    • OR
  • bool

    false if quest couldn't be abandoned

  • string

    Error message explaining the failure

Example Usage

local success, error = ix.quetes.Abandon(client, "repair_mission_01")
if success then
    client:Notify("Quest abandoned")
end

ix.quetes.Accept(client, questID)

 View source »

Attempts to assign a quest to a player, performing all necessary validation checks.

Validates class requirements, cooldowns, attempt limits, and assigns objective positions.

Parameters

  • client table

    The player entity attempting to accept the quest

  • questID string

    The unique quest identifier

Returns

  • bool

    true if quest was successfully accepted

    • OR
  • bool

    false if quest acceptance failed

  • string

    Error message explaining why the quest couldn't be accepted

Example Usage

local success, error = ix.quetes.Accept(client, "repair_mission_01")
if not success then
    client:Notify(error)
end

ix.quetes.IsQuestActiveByOtherPlayer(questID, excludeClient)

 View source »

Checks if a quest is currently active by another player (excluding the specified client).

Used primarily for exclusive quests to prevent conflicts.

Parameters

  • questID string

    The unique quest identifier

  • excludeClient table

    The client entity to exclude from the check

Returns

  • bool

    false if no other player has the quest active

    • OR
  • bool

    true if another player has the quest active

  • Player

    The player entity who has the quest active

    client:Notify(otherPlayer:Name() .. " is already on this quest")
end

Tables

QUEST_TYPES

 View source »

Table defining all available quest types with their visual properties and metadata.

Each quest type has a unique identifier (numeric constant) and contains display information used throughout the quest system UI. Quest types determine the category and nature of objectives.

Structure for each quest type:

  • name (string): Display name for the quest type
  • icon (Material): Icon material for visual display

Available quest types:

  • QUEST_REPARER (1): Repair-type quests for fixing equipment or infrastructure
  • QUEST_NETTOYER (2): Cleaning-type quests for clearing areas or removing hazards
  • QUEST_TUER (3): Elimination-type quests for combat objectives
  • QUEST_LIVRER (4): Delivery-type quests for transporting items or messages
  • QUEST_RECUPERER (5): Retrieval-type quests for collecting items or resources

Fields

  • [QUEST_REPARER]

  • icon

Example Usage

local questType = QUEST_TYPES[QUEST_REPARER]
print(questType.name) -- "Reparer"
surface.SetMaterial(questType.icon)

-- Iterate through all quest types
for typeID, typeData in pairs(QUEST_TYPES) do
    print(string.format("Type %d: %s", typeID, typeData.name))
end

ix.quetes.activePlayers

 View source »

Table tracking active players for each quest, indexed by quest ID and player Steam ID.

Used to prevent multiple players from taking exclusive quests simultaneously.

Structure:

  • [questID] (table): Table of Steam IDs mapped to boolean values for active players

Example Usage

if ix.quetes.activePlayers["repair_mission_01"]["STEAM_0:1:12345"] then
    print("Player is working on this quest")
end

ix.quetes.assignedPoints

 View source »

Table tracking assigned point indices for non-exclusive quests.

Prevents multiple players from being assigned to the same objective location.

Structure:

  • [questID] (table): Table mapping point indices to assignment data
  • [questID][pointIndex] (table): Assignment information
  • [questID][pointIndex].playerID (string): Steam ID of the assigned player
  • [questID][pointIndex].assignedTime (number): Unix timestamp of assignment

Example Usage

local assignment = ix.quetes.assignedPoints["cleanup_01"][3]
if assignment then
    print("Point assigned to:", assignment.playerID)
end
